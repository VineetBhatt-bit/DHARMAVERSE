# Contribution Rules

Every contribution to DHARMAVERSE should move the product closer to the Sentience Engine vision.

## Commit Standard

Each commit should do one clear thing:

- Add a product capability.
- Improve a user-facing experience.
- Strengthen the data model.
- Fix a verified issue.
- Improve documentation that guides implementation.

Avoid commits that only reshuffle code, rename files without purpose, or add unused assets.

## Milestone Discipline

Before starting a change:

- Identify the milestone it belongs to.
- Keep the scope small enough to verify.
- Prefer working UI or working data over abstract plans.

Before committing:

- Run available syntax or behavior checks.
- Verify the app locally when UI changes.
- Update milestone status when a milestone is completed.
- Make sure Git status is clean after the push.

## Product Bar

A DHARMAVERSE contribution should preserve these principles:

- Places should feel alive, not like database rows.
- Narrative, atmosphere, memory, and causality matter as much as layout.
- Ordinary people, local culture, ecology, language, and sound are first-class memory sources.
- Every feature should help answer why a place matters.
