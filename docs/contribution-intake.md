# Contribution Intake Console

The Contribution Intake Console is the first guided workflow for preparing new archive records before they become part of the DHARMAVERSE memory archive.

## Purpose

Archive contributions need more than a title and description. They need keeper credit, source context, consent status, cultural sensitivity, and a clear review rule. This console turns those requirements into a working interface attached to the active place.

## Current Fields

The intake draft includes:

- Active place
- Record type
- Sensitivity level
- Working title
- Keeper or source community
- Source note
- Consent and credit path status
- Required context for the selected record type
- Review rule for the selected sensitivity level

## Record Types

Current record types:

- Oral tradition
- Song or chant
- Local knowledge
- Ritual note
- Sound memory

Each type defines required context so contributors know what must be captured before a curator accepts the record.

## Sensitivity Levels

Current sensitivity levels:

- Public: can be shown after source and keeper credit are confirmed.
- Community review: needs keeper, family, institution, or local community review before publication.
- Restricted: remains private until permission, privacy, and cultural boundaries are resolved.

## Product Behavior

When a user changes the selected place, the intake draft resets for that place. When a user changes type, sensitivity, title, keeper, source, or consent status, the preview updates immediately. The preview is not stored yet; it is a review-ready draft surface for the future contribution backend.
